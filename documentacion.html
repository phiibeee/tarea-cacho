Estructuras y clases del motor del juego Cacho Alalay (C++ / Cacho Engine)

Introducción al Modelo

En el juego de Cacho (o Alalay), la toma de decisiones gira en torno a qué dados conservar, qué dados volver a lanzar y dónde anotar los resultados obtenidos. Para modelar este proceso de forma estructurada, el simulador del juego en C++ utiliza principalmente dos construcciones fundamentales definidas en Actuacion.h:

Anotacion

Representa el resultado concreto de haber elegido una de las categorías del tablero del Cacho. Define el tipo de juego seleccionado y los puntos asociados.

Estructura Simple

Actuacion

Representa una acción completa o jugada realizable en el turno de un jugador. Esto incluye decisiones tales como lanzar ciertos dados, anotar puntos en un casillero, tachar un casillero ("sobre" o directo), o declarar una dormida.

Clase con Comportamiento
Contexto del Cacho: Cada turno de un jugador consiste en hasta dos intentos independientes. En el primer intento del turno, un jugador puede anotar directamente una jugada, dejarla "sobre" la mesa para meter presión al siguiente jugador, o volver a lanzar algunos dados.

Estructura Anotacion

Anotacion es un struct plano que describe un puntaje asignado (o a asignar) en un determinado casillero de la cartilla o marcador del jugador.

Anotacion.h 
struct Anotacion {
    std::string juego;  // Nombre de la categoría de puntaje
    int puntos;         // Puntos obtenidos (-1 indica no aplica, 0 indica borrado)
};

Miembros de la Estructura

  • juego (std::string)
    Identificador de la jugada del tablero. Los valores posibles son:
    • Chicas: "balas" (unos), "tontos" (doses), "trenes" (treses), "cuadras" (cuatros), "quinas" (cincos), "senas" (seises).
    • Mayores: "escalera", "full", "poker", "grande" (primer grande), "grande2" (segunda grande).
    • Especiales: Categorías temporales con aclaraciones como "juego_name (de la 1ra mano)".
  • puntos (int)
    La puntuación correspondiente a la categoría.
    • Puntaje positivo: El puntaje calculado según las reglas (p. ej., 20/25 puntos para escalera, 30/35 para full, 40/45 para póker, 50 para grande, o el total sumado de dados repetidos para las chicas).
    • Valor 0: Significa que la categoría ha sido "tachada" o "borrada" en el tablero debido a una mala tirada.
    • Valor -1: Representa que la anotación no aplica a la jugada actual (por ejemplo, cuando la acción es "lanzar").

Clase Actuacion

La clase Actuacion empaqueta todos los detalles de una acción posible devuelta por el sistema de reglas del juego a través de la clase Anotador. Cada jugador (IA o humano) recibe un listado de estas actuaciones en su turno y debe seleccionar el índice de la actuación que desea ejecutar.

Actuacion.h 
class Actuacion {
public:
    std::string accion;             // Tipo de acción ("lanzar", "anotar", "sobre", "nada", "dormida")
    Anotacion anotacion;            // Estructura Anotacion asociada
    std::vector<int> indiceDados;   // Índices de los dados [0-4] involucrados en la acción
    bool seRespeta;                 // Indica si la acción es parte de respetar un sobremesa

    // Constructor
    Actuacion(std::string acc, Anotacion anot, std::vector<int> idx, bool respeta)
        : accion(acc), anotacion(anot), indiceDados(idx), seRespeta(respeta) {}

    void display() const;
    void displayCompacto(int indice = -1) const;
};

Miembros de la Clase

  • accion (std::string)
    Describe el comportamiento semántico de esta jugada:
    • "lanzar": El jugador desea volver a tirar los dados indicados en indiceDados.
    • "anotar": El jugador decide registrar la jugada en su tablero.
    • "sobre": Acción válida únicamente en el intento 1, donde el jugador anota en la mesa y le pasa los dados al rival para que decida "si respeta" la anotación del sobremesa.
    • "dormida": Declaración de victoria inmediata (dormida) al obtener 5 dados iguales del mismo valor de una sola tirada.
    • "nada": Comportamiento nulo o de control en turnos especiales (como segundas pasadas de tiro).
  • anotacion (Anotacion)
    Estructura que describe qué categoría del tablero y cuántos puntos se ven afectados por la acción (mayormente para "anotar", "sobre" o tachar).
  • indiceDados (std::vector<int>)
    Vector con valores en el rango [0-4] que indica cuáles dados están involucrados en la acción. En una acción de tipo "lanzar", este vector especifica exactamente cuáles dados volverá a tirar el motor del juego.
  • seRespeta (bool)
    Variable booleana. Si es true, se indica que la tirada actual respeta el juego del rival anterior o una jugada de mano especial. Si es false, no se aplica respeto automático.

Métodos Integrados

  • display() const
    Imprime los detalles de la actuación en múltiples líneas en la salida estándar std::cout. Útil para depuración y visualización de menús para jugadores humanos.
  • displayCompacto(int indice = -1) const
    Imprime la representación formateada en una sola línea del objeto. Si se pasa un valor positivo a indice, se antepone dicho valor entre corchetes, p. ej. [i]. Este formato es el utilizado principalmente por el motor de juego interactivo en consola.

Ejemplos Básicos de Uso en C++

A continuación se presentan diferentes ejemplos prácticos que ilustran cómo inicializar y manipular estos dos componentes en un flujo real de C++.

Cómo crear instancias directas de Anotacion y Actuacion usando inicializadores directos y constructores:

main.cpp 
#include "Actuacion.h"
#include <iostream>
#include <vector>

int main() {
    // 1. Instanciar una anotación directa de Full de Mano (35 puntos)
    Anotacion fullMano = {"full", 35};

    // 2. Crear una actuación para anotar el Full en la planilla
    Actuacion accionAnotar("anotar", fullMano, {}, false);

    // 3. Crear una actuación para lanzar los dados en las posiciones 0, 2 y 4
    Anotacion sinAnotar = {"", -1};
    Actuacion accionLanzar("lanzar", sinAnotar, {0, 2, 4}, false);

    // 4. Crear una actuación para tachar/borrar la categoría "grande"
    Actuacion accionTachar("anotar", {"grande", 0}, {}, false);

    return 0;
}

Ejemplo de cómo un Bot evaluador analiza el vector de Actuacion disponibles provistas por el motor de juego para tomar una decisión informada:

JugadorHeuristico.h 
#include "Jugador.h"
#include <iostream>
#include <vector>

class JugadorHeuristico : public Jugador {
public:
    JugadorHeuristico(std::string nom) : Jugador(nom) {}

    int jugar(const std::map<std::string, Marcador>& /*marcadores*/,
              const std::vector<Actuacion>& actuacionesPosibles,
              const std::vector<int>& /*dados*/,
              const Anotacion& /*resultadoPrevio*/) override {
        
        // Estrategia heurística básica:
        // 1. Si existe una victoria inmediata ("dormida"), tomarla inmediatamente.
        for (size_t i = 0; i < actuacionesPosibles.size(); ++i) {
            if (actuacionesPosibles[i].accion == "dormida") {
                std::cout << nombre << " declara DORMIDA y gana!" << std::endl;
                return i;
            }
        }

        // 2. Si hay una opción para anotar una jugada de alto valor (>= 30 pts) de manera directa
        for (size_t i = 0; i < actuacionesPosibles.size(); ++i) {
            if (actuacionesPosibles[i].accion == "anotar" && 
                actuacionesPosibles[i].anotacion.puntos >= 30) {
                return i;
            }
        }

        // 3. De lo contrario, por defecto volver a lanzar dados (generalmente la primera opción de lanzamiento)
        for (size_t i = 0; i < actuacionesPosibles.size(); ++i) {
            if (actuacionesPosibles[i].accion == "lanzar") {
                return i;
            }
        }

        // 4. Retorno de emergencia (último elemento disponible)
        return actuacionesPosibles.size() - 1;
    }
};
